# Friendship Signals in Telegram

Исследовательский Telegram Web App, который превращает экспорт личного диалога `result.json` в воспроизводимый анализ ритма общения, взаимности, лексики и структуры ответов. Проект выполнен в рамках курса по digital humanities.

## Исследовательский вопрос

**Насколько наблюдаемые характеристики личной Telegram-переписки — частота сообщений, баланс участия, время ответа, инициирование диалогов и тематическая лексика — позволяют описать изменение интенсивности и взаимности общения во времени?**

В проекте не утверждается, что дружбу можно объективно измерить одним числом. «Индекс близости» — интерфейсная сводка нескольких признаков, а интерпретируются прежде всего отдельные показатели и их динамика.

## Новизна

Проект объединяет в одном воспроизводимом пайплайне:

- парсинг стандартного Telegram JSON;
- очистку смешанного поля `text` регулярными выражениями;
- временную агрегацию в `pandas`;
- частотный анализ и TF-IDF по участникам;
- ориентированный граф переходов между авторами сообщений;
- осторожную LLM-интерпретацию агрегатов;
- Telegram Web App как интерактивную форму представления результатов.

Новизна состоит не в изобретении отдельной метрики, а в адаптации комплекса методов digital humanities к приватному диалогу и в разделении вычисляемых наблюдений и интерпретаций.

## Предыдущие исследования

Теоретической отправной точкой служит понятие силы социальной связи у Марка Грановеттера. В цифровой среде Gilbert и Karahalios показали, что силу связи можно приближенно моделировать по наблюдаемым признакам коммуникации, но такая модель остается вероятностной. Работа Jeffrey Hall подчеркивает роль времени и совместного взаимодействия в формировании дружеских отношений.

1. Granovetter, M. S. (1973). *The Strength of Weak Ties*. American Journal of Sociology, 78(6), 1360–1380. [DOI](https://doi.org/10.1086/225469).
2. Gilbert, E., & Karahalios, K. (2009). *Predicting Tie Strength With Social Media*. CHI '09. [DOI](https://doi.org/10.1145/1518701.1518736).
3. Hall, J. A. (2019). *How many hours does it take to make a friend?* Journal of Social and Personal Relationships, 36(4), 1278–1296. [DOI](https://doi.org/10.1177/0265407518761225).
4. Telegram. *Chat Export Tool, Better Notifications and More* (описание официального экспорта данных). [Источник](https://telegram.org/blog/export-and-more).

## Данные

Основной материал исследования — экспорт одного диалога из Telegram Desktop в формате JSON. Единица анализа — текстовое сообщение. Для каждого сообщения используются:

| Поле | Назначение |
|---|---|
| `id` | идентификатор сообщения |
| `date` | временная метка |
| `from` | автор |
| `text` | текст или массив текстовых сущностей |
| `reply_to_message_id` | явная связь ответа, если присутствует |

Из-за приватности реальная переписка не публикуется. В репозитории находится небольшой синтетический датасет [data/example_result.json](data/example_result.json), воспроизводящий структуру Telegram JSON. Для финальных чисел необходимо выполнить тетрадку на обезличенном экспорте автора и указать в отчете его объем и временной диапазон.

## Метод

1. **JSON и regex:** извлечение сообщений; удаление URL, упоминаний и служебных символов; нормализация регистра и пробелов.
2. **Pandas:** сортировка по времени, подсчет сообщений, длины, вопросов, стартов после паузы в 6 часов, агрегация по дням и месяцам.
3. **Частотный анализ и TF-IDF:** поиск общей лексики и слов, характерных для каждого участника.
4. **Сетевой анализ:** ориентированный взвешенный граф `автор предыдущего сообщения → автор следующего сообщения`.
5. **LLM:** генерация текстовой интерпретации только из агрегатов. Модель получает инструкцию не ставить диагнозы и не приписывать скрытые мотивы.

Метод подходит для вопроса, потому что сочетает временные, лексические и структурные признаки. Ни один показатель не трактуется как самостоятельное доказательство дружбы.

## Пилотные результаты

На синтетическом примере пайплайн корректно:

- обработал 24 сообщения за период с 5 января по 20 апреля 2024 года;
- показал равный вклад участников — 12 и 12 сообщений, коэффициент взаимности 1,0;
- обнаружил 5 и 4 начала новых эпизодов после паузы более шести часов;
- оценил среднее время ответа в 63,6 и 8,3 минуты, то есть выявил различие даже при равном объеме сообщений;
- показывает динамику активности по дням и месяцам;
- выделил повторяющиеся слова «спасибо», «давай», «проект» и специфичную лексику каждого автора;
- строит направленные переходы между репликами.

Численные результаты синтетического примера не являются содержательным выводом о дружбе. Исследовательская интерпретация должна основываться на реальном обезличенном корпусе и сравнении нескольких временных периодов.

## Ограничения

- Telegram-экспорт не содержит прочтений, удаленных сообщений и полного контекста офлайн-отношений.
- Быстрый ответ может зависеть от расписания, часового пояса или срочности, а не от близости.
- Частота сообщений не равна эмоциональной близости.
- TF-IDF показывает различимость слов, но не их смысл в конкретном контексте.
- Граф переходов приближает структуру ответов и не всегда совпадает с семантической адресацией.
- LLM может создавать убедительные, но неподтвержденные объяснения; поэтому ей передаются агрегаты, а результат маркируется как интерпретация.
- Исследование одного диалога является case study и не обобщается на всех пользователей Telegram.

## Перспективы

- сравнить несколько типов отношений на одинаковом наборе признаков;
- добавить тематическое моделирование по периодам;
- отдельно анализировать реакции, медиа и голосовые сообщения;
- провести ручную разметку эпизодов и проверить согласованность с автоматическими метриками;
- заменить фиксированные веса индекса на эмпирически валидированную модель.

## Воспроизведение исследования

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
jupyter lab notebooks/research.ipynb
```

По умолчанию тетрадка читает `data/example_result.json`. Для личного корпуса измените переменную `DATA_PATH`, не добавляя приватный файл в Git.

## Запуск Telegram Web App

```bash
cp .env.example .env
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Для запуска внутри Telegram нужен публичный HTTPS URL, например туннель `ngrok http 8000`. Его необходимо записать в `WEB_APP_URL`, а затем запустить минимального бота-кнопку:

```bash
python3 -m app.bot
```

DeepSeek опционален. Без `DEEPSEEK_API_KEY` приложение показывает все вычисляемые результаты и локальное текстовое резюме.

## Этика и приватность

Перед анализом необходимо получить согласие участника переписки. Имена следует заменить псевдонимами. Приватные данные и `.env` нельзя публиковать в GitHub. Внешней LLM отправляются только агрегированные характеристики, но пользователь все равно должен осознавать факт внешней обработки.
