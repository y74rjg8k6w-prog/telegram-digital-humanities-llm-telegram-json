# План защиты проекта на 5–7 минут

## 1. Проблема и вопрос — 45 секунд

- Личные чаты создают подробный цифровой след отношений, но частота сообщений сама по себе не равна дружбе.
- Исследовательский вопрос: насколько временные, лексические и сетевые признаки позволяют описать изменение интенсивности и взаимности общения?
- Объект — один обезличенный диалог Telegram; единица анализа — сообщение.

## 2. Данные и этика — 45 секунд

- Показать структуру `result.json`: `date`, `from`, `text`, `reply_to_message_id`.
- Назвать реальный объем корпуса и временной диапазон.
- Объяснить псевдонимизацию, согласие участника и почему сырые приватные данные не размещены в GitHub.

## 3. Метод — 90 секунд

- JSON + regex: парсинг и очистка.
- Pandas: временные группы, баланс, вопросы, паузы, время ответа.
- Частоты и TF-IDF: общие и характерные темы.
- NetworkX: ориентированный граф переходов между авторами.
- LLM: только интерпретация агрегатов с запретом на диагнозы.

## 4. Демонстрация продукта — 90 секунд

- Открыть Telegram Web App.
- Загрузить заранее подготовленный обезличенный JSON.
- Показать четыре ключевых элемента: временной график, баланс, слова/TF-IDF, граф диалога.
- Показать, что без API-ключа вычислительный анализ продолжает работать.

## 5. Результаты — 60 секунд

Заменить пункты ниже на факты из реального корпуса:

- самый активный и самый тихий период;
- насколько сбалансировано участие;
- кто чаще начинает эпизоды после паузы;
- какие темы общие, а какие характерны каждому;
- совпадают ли разные признаки или дают противоречивую картину.

## 6. Ограничения и вывод — 45 секунд

- Метрики описывают переписку, а не внутреннее состояние людей.
- Один кейс нельзя обобщать на всех пользователей.
- Новизна проекта — общий воспроизводимый пайплайн и интерактивный Web App.
- Следующий шаг — ручная разметка эпизодов и сравнение нескольких типов отношений.

## Что подготовить перед экзаменом

1. Вписать в README реальный объем и даты корпуса.
2. Выполнить notebook на обезличенном реальном файле и сохранить выводы.
3. Сделать 5–6 слайдов по структуре выше.
4. Проверить репозиторий в чистом окружении.
5. Подготовить резервные скриншоты Web App на случай проблем с интернетом.
